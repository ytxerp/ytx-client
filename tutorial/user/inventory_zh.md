# YTX 存货

大家好，今天介绍一下 **YTX** 是如何管理存货的。

当前软件版本是 **0.2.7**，下载地址在 [Gitee](https://gitee.com/ytxerp/ytx-client/releases/tag/0.2.7)，链接也会贴在评论区。

---

## 主界面

首先，来看一下主界面。

1. **登录**
   - 登录时需要提供 **邮箱**、**密码** 和 **工作区**。
   - 邮箱和密码是用户自已注册的，工作区是管理员创建的；用户注册成功后可以申请加入某个工作区，然后管理员设置相应的权限、放行后才可以访问数据。
   - 可以选择 **是否本机保存密码**，方便后续的登录。

2. **菜单**
   - 顶部是 **菜单栏**，有：文件、编辑、账号三组。
   - 接着是 **工具栏**，依次是：插入节点、追加节点、编辑节点名称、重置节点颜色、创建节点组、追加记录、删除、全选、反选、空选、账单、结算。
   - 左侧是 **版块栏**，当前开放了：财务、任务、存货、合作。

3. **存货**
   - 今天要介绍的是存货。存货，可能会有不同的称呼，比如物品、产品、仓库等，其实都是同一个意思。
   - 默认情况下，界面是空的，需要通过创建**节点**来逐步搭建结构。

---

## 属性

接下来介绍节点的**核心属性**，打开追加节点界面

- **规则**。
  可以设置为 `DICD` 或 `DDCI`，这会决定节点余额的计算方式。

- **类型**
  - **L，叶节点**，是最终录入数据的地方。
  - **B，枝节点**，它会对名下叶节点数据进行汇总。

- **单位**
  - **INT，内部编号**，自己公司的产品编号。
  - **EXT，外部编号**，客户的采购编号或供应商的销售编号；开单时，会根据自有编号、自动引入客户的编号。
  - **POS，位置**，表示产品存放的位置。
  - 实际使用中，内部编号和位置最常用；外部编号是用来映射关系的、在**合作**版块中设置。

## 演示

下面通过一个小例子，看看存货版块是如何工作的。

### 节点

1. 添加一个产品枝节点
   - 快捷键：`Alt + P`
   - 命名为 `苹果`，类型 `B`，单位 `INT`。

2. 追加三个产品子节点
   - 选中 `苹果`，按 `Alt + P`。
   - 分别命名为：`手机`、`电脑` 和 `平板`，类型为 `L`，单位同样是 `INT`。

3. 添加一个位置枝节点
   - 选中 `苹果`，按 `Alt + N`，同层插入。
   - 命名为 `位置`，类型为 `B`，单位设置为 `POS`。

4. 追加三个位置子节点
   - 选中 `位置`，按 `Alt + P`。
   - 分别命名为：`1号箱`、`2号箱` 和 `3号箱`，类型 `L`，单位 `POS`。
5. 调整位置节点的规则，和产品节点相反就好
6. 可以自行添加其他属性

简单的结构就建完了，实际场景中肯定会使用更多的节点，按需求添加就好。

### **注意**

- 追加节点，会在选中的枝节点后、追加子节点；如果没有选中，就会追加到根节点，也就创建了顶层节点。
- 插入节点，会在选中的节点下、插入新节点；如果没有选中，操作会无效。
- 节点的**类型**和**单位**一旦设定，就不能修改，如果填错了，得删除重新建。
- 枝节点只会对**单位**相同的叶节点进行数据汇总。
- **位置**节点的单位必须准确，在订单版块开单时，才不会收集到错误的数据。

### 记录

接下来是录入数据。

1. **界面**
   - 双击`手机`，进入登记数据界面。
   - 工具栏可以使用的功能会自动高亮。

2. **记录**
   - 新建一条记录：`Ctrl + N`。
   - 关联`3号箱`。
   - 借方`50`，成本`7749`。

3. **属性**
   - **描述**：登记记录关键信息，搜索时会用到。
   - **D**：用来添加文档，这里只会保存文件的相对路径、相对于`首选项-文件夹`的路径，不会同步文件自身。
   - **S**：表示状态，经常用于核对。
   - **关联**：可以关联到 **位置** 或 **产品**。
     - 关联位置，就表示存放地点。
     - 关联产品，可以表示产品从一个仓库挪到另一个仓库。
   - **借方、贷方**：
     - 借方和贷方用于登记数据，并且一定成对出现。
     - 当前节点借方有数据，关联节点贷方一定有同样的数据；当前节点贷方有数据，关联节点的借方也一定有同样的数据。
     - `Ctrl + J`，跳转到关联节点界面看一下，贷方有`50`。
     - 双击标题，刚刚录入的数据已经同步到节点中了。
   - **余额**：
     - 节点规则是 `DICD`，余额 = 借方 − 贷方。
     - 节点规则是 `DDCI`，余额 = 贷方 − 借方。

4. **查看**
   - 在 `3号箱` 中查看里面具体有哪些东西。
   - 在 `手机` 中查看它存放在哪些个位置。

5. **搜索**
   - `Ctrl + F`，可以通过**节点名称**或**记录描述**搜索。
